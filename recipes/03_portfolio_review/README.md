# Recipe 03: Weekly Portfolio Review

> An agent that ingests your paper-trading positions and writes a weekly review — 3 observations, 3 questions for next week.

## What this does

Once a week, the agent pulls your paper-trading positions, recent trades, and cash balance from Longbridge. It computes basic exposure metrics (sector mix, top-3 concentration, week-over-week PnL, win rate of closed trades), then produces a structured journal-style Markdown review: **3 observations** about the past week + **3 open questions** for next week.

Primary trigger: paste the prompt into your agent on Friday EOD or Sunday evening. Headless trigger: `python main.py` from a cron. Output: Markdown printed to chat (or written to `reviews/YYYY-Www.md` in the headless path).

## Quickstart — MCP-first

**You need:** Claude Desktop (or Cursor / Claude Code / Codex) with the Longbridge MCP installed. See [`MCP_SETUP.md`](../../MCP_SETUP.md).

Open your agent and paste:

```
Using the Longbridge MCP, write my weekly portfolio review:

1. Pull my paper-trading account state: NAV, cash, and all stock positions.
2. Pull every execution from the last 7 days (or this ISO week, whichever you can).
3. Compute these metrics:
     - Trade count and win rate of closed trades
     - Average holding period
     - Largest realized winner and loser this week
     - Sector mix of current positions (GICS sector via security metadata)
     - Top-3 position concentration as % of NAV
     - Week-over-week NAV change
     - Largest unrealized winner and loser among open positions
4. Write a Markdown review with this exact structure:
     # Week of <ISO week, e.g. 2026-W19>
     ## Stats
     <one-line summary: NAV, WoW%, # trades, win rate>
     ## 3 observations
     <three numbered bullets — terse, first-person, no hedging>
     ## 3 questions for next week
     <three numbered bullets — what to investigate, not what to do>

Voice: journal-style. Short. Opinionated. No "consider" or "you might want to."
This is for me, not for a client.

Paper-trading account only. No order placement.
```

## Walkthrough — what the agent will do

The MCP tool-call sequence:

1. **`account.balance`** — total NAV and cash.
2. **`account.positions`** — every open stock position with cost basis and current value.
3. **`account.executions(start=last_monday, end=now)`** — every fill in the window.
4. **`security.meta` (per held ticker)** — pulls industry → rolled up to GICS sector for the sector-mix metric.
5. **`account.nav_history(days=14)`** — gives the agent enough to compute WoW NAV change.
6. **Compute step (no tool call)** — agent does the arithmetic for win rate, concentration, holding period, and the unrealized winner/loser ranks.
7. **Synthesis** — Markdown review with the exact section structure you asked for.

Expected output: a single Markdown block with `# Week of YYYY-Www`, a one-line stats summary, then 3 observations and 3 questions.

## What you can change

- **Review window** — change "last 7 days" to "last 30 days" for a monthly cadence, or "since I last asked" if your agent has memory.
- **Observation/question count** — default is 3+3. Change to 5+5, or split into "what worked / what didn't / what I'll try."
- **Sector taxonomy** — instead of GICS, ask the agent to roll positions into your own buckets ("AI compute / Chinese consumer / HK property / cash"). Pure prompt edit.

## Automated / headless variant

For a Sunday-night cron or a GitHub Action:

```bash
# From the repo root
cp .env.example .env
pip install longbridge anthropic
cd recipes/03_portfolio_review
python main.py
```

The Python fallback writes to `recipes/03_portfolio_review/reviews/YYYY-Www.md` (ISO week), and prints the same content to stdout. If a file for the current week already exists, it appends a delta section instead of overwriting.
