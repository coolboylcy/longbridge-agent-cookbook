# Recipe 01: Earnings Monitor

> An agent that watches your Longbridge watchlist for upcoming earnings and emits a 24h pre-brief plus a 1h post-recap.

## What this does

The agent reads your Longbridge watchlist, identifies symbols with earnings within the next 24 hours, and writes a concise brief per ticker: prior-quarter results, consensus estimates, options-implied move, and 3 things to watch. One hour after the earnings release, a second pass produces a recap comparing reported numbers against consensus and the implied move.

Primary trigger: paste the prompt below into Claude Desktop / Cursor / Codex. Headless trigger (cron, GitHub Actions): `python main.py`. Output: Markdown briefs, one per ticker.

## Quickstart — MCP-first

**You need:** Claude Desktop (or Cursor / Claude Code / Codex) with the Longbridge MCP installed. If you haven't, see [`MCP_SETUP.md`](../../MCP_SETUP.md) — about 60 seconds.

Open your agent and paste:

```
Using the Longbridge MCP, do an earnings sweep:

1. Pull my watchlist.
2. For every ticker with an earnings release in the next 24 hours, fetch:
   - Last 4 quarters' EPS and revenue (actual vs. consensus)
   - Current consensus EPS and revenue estimate
   - The front-week options chain — compute the implied move as
     (ATM call mid + ATM put mid) / spot
   - The past 8 quarters' beat/miss pattern
3. For each upcoming earner, write a tight Markdown brief with sections:
   - Prior quarter recap (1 line)
   - Consensus this quarter (1 line)
   - Implied move (1 line)
   - Beat/miss history (1 line)
   - Three things to watch (3 bullets)

If no tickers have earnings in the next 24h, say so and stop.
```

For the **post-earnings recap** (run ~1h after release), use this follow-up prompt:

```
Using the Longbridge MCP, for [TICKER] which just reported:

1. Pull the reported revenue, EPS, and any guidance update.
2. Compare against the consensus you cited earlier and the implied move.
3. Write one paragraph: "what changed."
4. End with a verdict line: beat / in-line / miss on revenue, EPS, guidance.
```

## Walkthrough — what the agent will do

The teaching part. When you paste the brief prompt, expect this MCP tool-call sequence:

1. **`watchlist.list`** — pulls your tracked tickers. The agent now knows the universe.
2. **`calendar.events` (per ticker, or batched)** — filters to events of type `earnings` within the next 24h. Anything outside the window drops out.
3. **`fundamentals.quarterly` (per upcoming earner)** — last 4 quarters of reported EPS/revenue.
4. **`fundamentals.consensus`** — current sell-side consensus for the upcoming quarter.
5. **`options.chain_expiries` + `options.chain` (front-week ATM)** — agent picks the nearest weekly expiry, identifies the ATM strike, reads the call+put mids, computes implied move.
6. **`fundamentals.beat_miss_history`** — last 8 quarters' surprise pattern.
7. **Synthesis (no tool call)** — the agent assembles the brief in Markdown and prints it back to chat.

Expected output: one Markdown block per ticker with earnings in the next 24h, or `No upcoming earnings in the next 24h.` if your watchlist is quiet.

## What you can change

- **Window size** — change "next 24 hours" in the prompt to "next 7 days" for a weekly look-ahead, or "next 4 hours" for a same-session sweep.
- **Brief sections** — drop the implied-move bullet, add a "macro context" paragraph, or switch the voice from neutral to opinionated. Edit the prompt; no code change required.
- **Output format** — ask for JSON instead of Markdown if you're piping into another tool, or "a single tweet per ticker" if you're drafting social.

## Automated / headless variant

For a cron job, GitHub Action, or any agent-free context:

```bash
# From the repo root
cp .env.example .env
# edit .env → LONGBRIDGE_APP_KEY / APP_SECRET / ACCESS_TOKEN, TRADING_MODE=paper
pip install longbridge anthropic
cd recipes/01_earnings_monitor
python main.py             # pre-brief mode
python main.py --mode recap
```

The Python fallback uses the Longbridge SDK directly and produces equivalent Markdown.
See `main.py` — it mirrors the same tool-call sequence the MCP agent would make.
